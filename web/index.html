        Historical heat map

        This layer is a screen‑space kernel density estimate (KDE) of the historical occurrence point set P. Each occurrence i contributes a radially symmetric kernel K with pixel radius r; heat at a screen pixel x is the sum of kernels weighted by per‑point intensity.

        Value(x) ≈ Σi∈P w_i · K(||x − x_i|| / r)
K: radial kernel (circular)
r: kernel radius in pixels (here radius≈18px, blur≈15px)
w_i: point weight (intensity)

        Data and weights:

        • Input: web/data/heat_all_r6.geojson with Point features.
        • Each feature contributes [lat, lng, intensity] to Leaflet.heat.
        • Intensity is log‑compressed to avoid a few dense points overwhelming the map.
        // map.js (excerpt)
const features = geo.features;
const step = features.length > 50000 ? Math.ceil(features.length / 50000) : 1; // downsample
for (let i = 0; i < features.length; i += step) {
  const f = features[i];
  if (!f || f.geometry?.type !== 'Point') continue;
  const [lng, lat] = f.geometry.coordinates;
  const count = (f.properties && typeof f.properties.count === 'number') ? f.properties.count : 1;
  const intensity = Math.max(0.2, Math.min(1, Math.log(1 + count) / 3));
  points.push([lat, lng, intensity]);
}
L.heatLayer(points, {
  radius: 18, blur: 15, maxZoom: 11, minOpacity: 0.25,
  gradient: { 0.2:'#1e3a8a', 0.4:'#2563eb', 0.6:'#f59e0b', 0.8:'#ef4444', 1.0:'#dc2626' }
});
        • Downsampling preserves hotspot shape; absolute amplitude can be scaled if needed.
        • Radius is in pixels, so smoothing adapts with zoom (coarser when zoomed out).
        • The heat map is global (all records); species selection and time chips do not affect it.
